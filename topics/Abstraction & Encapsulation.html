Abstraction vs Encapsulation in OOP

What is Abstraction

Abstraction refers to the concept of hiding the complexities of a system from the users of that system.

Abstraction is a general concept formed by extracting common features from specific cases.

Abstraction solves the problem in Design Level

Abstraction means hiding implementation using Abstrac class and Interface.

What is Encapsulation

Encapsulation is a language construct which bundles data and behavior together. In addition, it restricts access to this data and behavior bundled together.

Encapsulation solves the problem in Implementation Level.

Example of encapsulation: hiding data using getters and setters.

To summarise:

Abstraction hides complexity of the system by presenting to the user a simplified interface. Whereas encapsulation is all about bundling of related data and behaviour together while restricting/controlling access to them, and in the process partially or fully hiding these members.
